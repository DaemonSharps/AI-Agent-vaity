# Этап 05: Frontend Chat Shell

## Цель

Заменить стартовый Vite UI на базовый интерфейс чата: список чатов, текущий чат, история сообщений, поле ввода и черно-белая цветовая схема.

## Начало этапа

Начинать этап можно, если завершен этап 04:

- Backend предоставляет HTTP API чатов.
- Backend предоставляет `GET /api/chats/{chatId}/messages`.
- WebSocket endpoint уже есть, но на этом этапе можно еще не подключать отправку сообщений.
- `npm run build --prefix frontend` проходит.

## Задачи

### 1. Удалить демо weather UI

Удалить из frontend отображение weather forecast.

Не оставлять пользователю элементы, не связанные с чатом.

### 2. Описать frontend-типы

Добавить типы:

- `Chat`
- `Message`

Они должны соответствовать RFC:

```ts
type Chat = {
  id: string;
  title: string;
  createdAt: string;
  updatedAt: string;
};

type Message = {
  id: string;
  chatId: string;
  role: 'user' | 'assistant';
  content: string;
  thinking?: string;
  createdAt: string;
  status: 'pending' | 'sent' | 'streaming' | 'failed' | 'complete';
  order: number;
};
```

### 3. Реализовать API-клиент для HTTP endpoints

Frontend должен уметь:

- Получить список чатов.
- Создать чат.
- Переименовать чат.
- Удалить чат.
- Получить историю сообщений чата.

Использовать `VITE_API_URL`.

### 4. Реализовать layout

Интерфейс должен содержать:

- Боковую или верхнюю панель списка чатов.
- Кнопку создания нового чата.
- Выделение активного чата.
- Заголовок текущего чата.
- Область истории сообщений.
- Поле ввода сообщения.
- Кнопку отправки.

На этом этапе кнопка отправки может быть временно отключена или показывать сообщение, что WebSocket будет подключен на следующем этапе.

### 5. Реализовать управление чатами

Frontend должен:

- Создавать чат через `POST /api/chats`.
- Переключаться между чатами.
- Загружать историю текущего чата.
- Переименовывать чат.
- Удалять чат с подтверждением.
- Блокировать создание 11-го чата или показывать ошибку backend.

### 6. Реализовать черно-белую тему

Правила:

- Основные цвета: белый и черный.
- Дополнительные цвета только для ошибок, disabled-состояний и минимальных акцентов.
- UI должен работать на desktop и mobile.

## Конец этапа

Этап завершен, когда:

- Weather UI удален.
- Видны список чатов и текущий чат.
- Можно создать, переименовать, удалить и выбрать чат.
- История сообщений загружается из backend.
- Лимит 10 чатов корректно отображается пользователю.
- UI адаптируется под mobile viewport.
- Проект собирается.

## Проверки

Выполнить:

```powershell
npm run build --prefix frontend
dotnet build AI-Agent-vaity.sln
docker compose config
```

При запущенном compose вручную проверить:

- Открывается `http://localhost:3000`.
- Можно создать чат.
- Можно переключиться между чатами.
- После перезагрузки страницы список чатов загружается с backend.

## Стабильное состояние после этапа

Frontend уже выглядит как чат-приложение и использует HTTP API для управления чатами. Отправка сообщений через WebSocket подключается на следующем этапе.
